Introduction to Java

Java is a high-level, object-oriented, robust, and secure programming language developed by Sun Microsystems in 1995. It is now maintained by Oracle Corporation.

Java follows the principle of Write Once, Run Anywhere (WORA), which means compiled Java code can run on all platforms that support Java without recompilation.

Java is widely used in:

  • Enterprise applications (Banking, Insurance)
  • Android mobile applications
  • Web applications and backend services
  • Cloud-based systems

Features of Java

Object-Oriented: Everything in Java is treated as an object, enabling modular programming.

Platform Independent: Java programs run on JVM, making them independent of hardware.

Secure: Java provides strong memory management and no pointer usage.

Robust: Strong exception handling and garbage collection.

Multithreaded: Java supports concurrent execution.

Java Architecture (JDK, JRE, JVM)

JDK: Java Development Kit includes compiler, debugger, and development tools.

JRE: Java Runtime Environment provides libraries and JVM to run Java programs.

JVM: Java Virtual Machine converts bytecode into machine-level instructions.

Java Program Structure

A basic Java program consists of a class and main method.

class Main {
    public static void main(String[] args) {
        System.out.println("Hello Java");
    }
}

The main method is the entry point of execution.

Variables in Java

A variable is a container used to store data values.

  • Local Variable: Declared inside a method.
  • Instance Variable: Belongs to an object.
  • Static Variable: Shared among all objects.

Data Types in Java

Primitive data types store simple values like int, float, char.

Non-primitive data types store references like String, Array, Objects.

Control Statements

Control statements decide the flow of program execution.

  • Decision making: if, if-else, switch
  • Loops: for, while, do-while
  • Jump: break, continue

Arrays in Java

An array stores multiple values of same data type. 

int[] arr = {10,20,30};

Arrays are zero-indexed and have fixed size.

Strings in Java

Strings are immutable in Java. Once created, their value cannot be changed.

StringBuffer and StringBuilder provide mutable strings.

Object-Oriented Programming

OOP is a programming paradigm based on objects and classes.

  • Encapsulation – Data hiding
  • Inheritance – Code reuse
  • Polymorphism – Multiple forms
  • Abstraction – Hide implementation

Inheritance

Inheritance allows one class to acquire properties of another class using the extends keyword.

It supports code reusability and hierarchical classification.

Polymorphism

Polymorphism means many forms.

Method Overloading (compile-time) and Method Overriding (runtime) are two types.

Abstraction

Abstraction hides implementation details and shows only essential features.

Achieved using abstract classes and interfaces.

Exception Handling

Exceptions are runtime errors that disrupt normal flow.

Handled using try, catch, finally, throw, throws.

Collections Framework

Collections store and manipulate groups of objects.

  • List – ArrayList, LinkedList
  • Set – HashSet, TreeSet
  • Map – HashMap, TreeMap

Multithreading

Multithreading allows concurrent execution of two or more threads.

Improves CPU utilization and performance.

JDBC

JDBC connects Java applications with databases.

Steps: Load driver → Create connection → Execute query → Close connection

Java 8 Features

Java 8 introduced functional programming concepts.

  • Lambda Expressions
  • Streams API
  • Functional Interfaces
  • Optional Class
